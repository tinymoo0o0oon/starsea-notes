## 产品骨架：留存飞轮

不是孤立功能堆叠，是一个因果闭环：

```
愿意写（体验+激励） → 经常打开 → 打开就逛（消费）
        ↑                                    ↓
   价值感不递减 ← 结构越来越好 ← 发现连线 + 手动连线
```

引擎在"逛"不在"写"——多数笔记 app 死于"写完不看"。
- **激励（彩蛋/许愿）** = 点火飞轮的火花塞。
- **连线/聚类/Obsidian** = 飞轮本体，让价值不随新鲜感衰减。

## 核心定位（2026-06-28 拍板）

**Obsidian 为核，向量为兜底。**

- 手动连线 = 主结构、**强弹簧**，聚类必须服从。
- 向量连线 = 只在你还没连的地方兜底、**弱弹簧**，信号弱就让它弱。
- 定位演化：原"你永不手动连线" → 现"连线是被鼓励的核心动作，向量只保证你永不被卡住"。
  手动连线从"不用做"升级为"主动做、且做了有奖励"。

## 待你定义的 open question

**#5 许愿"在某种规则下实现"指啥？两种解读，成本差一个数量级：**
- **A 世界许愿（装饰）**：`<这个星球长满仙人掌>` → 改星球外观/世界氛围，是 #7 文生图的入口。
- **B 功能许愿（开发者愿望箱）**：`<我希望能手动连线>` → 写进愿望日志，开发读到后实现。
- 可并存（同一语法按内容路由），轮到 #5 再定。

**#5 随机飞船 · 重新消费老笔记**（用户 2026-06-28 提，与许愿同量级）：
- 一艘随机小飞船随机降落到某颗星上，把你引回那条老笔记，对抗"写完就不看"。
- 加权随机：被选中概率 ∝ **越久没打开**（时间衰减）× **字数越多**（份量大的更值得回看）。
- 前置：现在 note 只存 `created`，没有"上次打开时间"。需先在落地阅读某条时记录 `lastOpened`，飞船权重才能算。
- 属飞轮"经常消费"一环，serendipity（偶遇）型激励，和彩蛋/许愿同簇。
